### 本月工作总结

本月主要围绕 buddy-mlir 项目开发环境的和调整 buddy-example 仓库的项目结构以方便下一步统一支持 RISC-V 的不同 DSA 后端展开工作。

针对 Nix 配置长期未更新导致的兼容性问题（仍停留在 LLVM 17 版本，且与新依赖如 nanobind 存在冲突），完成了 Nix 配置的升级工作，使其支持最新版本的 LLVM 和相关依赖，便于 buddy-examples 仓库后续集成不同后端环境。该变更通过 `nix build .#buddy-llvm` 和 `nix build .#buddy-mlir` 验证。

此外，为解决 buddy-examples 中端到端测试用例缺乏性能剖析工具的问题 进行了 RVProf Dialect 的开发。该 Dialect 针对 RISC-V DSA 设计，支持基于硬件周期（rdcycle）的性能剖析，包含 Dialect 定义、Pass、运行时库支持以及相关构建配置。

同时，在 buddy-examples 仓库中开启了仓库结构重构工作，旨在更好地支持多种 DSA 后端。具体包括拆分 FireSim 与 Chipyard 目录、增加 Bebop 支持更多仿真工具，并将 Bebop 集成至 Nix flake 配置。该重构依赖 buddy-mlir 中的相关提交。

### 本月提交的PR

-[[Midend/RVProf] feat: add a profiling dialect for riscv DSAs](https://github.com/buddy-compiler/buddy-mlir/pull/781) 为 RISC-V DSA 实现性能剖析 Dialect，支持基于周期计数的硬件性能分析，包含 Dialect 定义、转换 Pass、运行时库及测试用例

-[[WIP] Restructure repository to support different DSA backends](https://github.com/buddy-compiler/buddy-examples/pull/7) 本月重构了 buddy-examples 仓库结构，实现独立的开发环境安装。拆分 [fireSim](https://github.com/firesim/firesim) 与 [chipyard](https://github.com/ucb-bar/chipyard) 项目，摆脱原先使用 chipyard 进行开发环境安装的依赖，使得后续可以拓展支持其他后端的开发环境。扩展 Bebop 支持并集成至 Nix 配置，提升对多种 DSA 后端的模块化支持。下个月将对接 [buckyball](https://github.com/DangoSys/buckyball) 作为新的DSA后端。
